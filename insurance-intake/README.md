# AJF 오프라인 매장 보험접수 현황

마이딜러(crm-dashboard)와 **완전히 별개의 독립 프로젝트**입니다. 로그인 시스템,
배포, 데이터 저장소 모두 마이딜러와 공유하지 않습니다.

오프라인 매장 고객 설문(별도 운영중인 신청 페이지)으로 접수된 건을 확인하고,
보험사업부가 상담 진행상황(중복보장점검완료일, 키트수령안내여부, 상담상태,
보험가입여부 등)을 기록하는 웹 대시보드입니다. 이미 운영중이던 두 개의 구글
스프레드시트를 그대로 데이터 원본으로 사용합니다.

- **보험사업부 시트** — "오프매장 전환DB_replit" 탭. 고객 설문 페이지가
  A~P열(이름, 연락처, 주민번호, 지점, 선택 선물 등)에 직접 기록하고,
  보험사업부가 그 뒤 칼럼에 진행상황을 기록합니다.
- **매장 시트** — "신청인원 현황" 탭. 보험사업부 시트 내용을 대부분 그대로
  가져와 보여주고(IMPORTRANGE), "고객 선물 지급일자 / 키트 불출인원 / 특이사항"
  3개 칼럼만 매장 직원이 직접 입력하도록 되어 있습니다. 이 값은 다시
  보험사업부 시트로 자동 반영됩니다.

이 문서는 **코딩을 전혀 모르는 분도 그대로 따라 하면 서비스를 띄울 수 있도록**
순서대로 작성했습니다.

---

## 0. 준비물 체크리스트

- [ ] 구글 계정 (두 스프레드시트에 접근 가능한 계정)
- [ ] 보험사업부 시트, 매장 시트 (이미 운영중)
- [ ] GitHub 계정 (이 코드가 들어있는 저장소에 접근 가능해야 함)
- [ ] Vercel 계정 (무료 플랜으로 충분, GitHub 계정으로 가입 가능)

## 1. 구글 클라우드 서비스 계정 만들기

마이딜러(crm-dashboard)에서 이미 서비스 계정을 만들어 쓰고 있다면 **그 계정을
그대로 재사용**해도 됩니다(같은 서비스 계정이 여러 스프레드시트에 공유될 수
있습니다). 새로 만들려면:

1. https://console.cloud.google.com/ 접속 후 구글 계정으로 로그인합니다.
2. 새 프로젝트를 만듭니다(예: `ajf-insurance-intake`).
3. "API 및 서비스" > "라이브러리"에서 **Google Sheets API**를 사용(Enable)으로
   설정합니다.
4. "API 및 서비스" > "사용자 인증 정보" > "사용자 인증 정보 만들기" >
   "서비스 계정"을 만듭니다.
5. 생성된 서비스 계정 > "키" 탭 > "키 추가" > "새 키 만들기" > JSON 선택 →
   JSON 파일이 다운로드됩니다. **안전하게 보관하세요.**
6. 서비스 계정 이메일 주소(`xxxxx@xxxxx.iam.gserviceaccount.com`)를 복사해둡니다.
7. JSON 파일 안의 `client_email`, `private_key` 값을 이후 환경변수 설정에 씁니다.

## 2. 두 스프레드시트를 서비스 계정과 공유하기

1. **보험사업부 시트**(`AJI_오프매장_DB신청_보험사업부 조회용`)를 열고
   **공유** 버튼 → 서비스 계정 이메일을 **편집자(Editor)**로 추가합니다.
2. **매장 시트**(`AJI_오프매장_DB신청_오프매장 조회용`)도 동일하게 서비스
   계정 이메일을 편집자로 공유합니다.

공유가 안 되어 있으면 앱이 데이터를 읽거나 쓰지 못합니다.

## 3. 보험사업부 시트에 "계정관리" 탭 만들기

로그인 계정 정보를 보험사업부 시트 안의 새 탭에서 직접 관리합니다. 마이딜러와
달리 매니저별 개인 시트가 없으므로 칼럼이 하나 적습니다.

1. **보험사업부 시트** 하단에 새 탭을 추가하고 이름을 정확히 `계정관리`로
   지정합니다.
2. 1행(헤더)에 아래 7개 컬럼명을 **A1부터 순서대로** 입력합니다.

   | A | B | C | D | E | F | G |
   |---|---|---|---|---|---|---|
   | 아이디 | 비밀번호 | 이름 | 권한 | 비고 | 실패횟수 | 잠김여부 |

3. 2행부터 계정을 한 줄씩 입력합니다. (F, G열은 비워두면 앱이 자동으로
   0/빈칸으로 채웁니다.)

   - **관리자(보험사업부) 계정 예시**

     | 아이디 | 비밀번호 | 이름 | 권한 | 비고 |
     |---|---|---|---|---|
     | admin | (원하는 비밀번호) | 보험사업부 | 관리자 | |

   - **매니저(매장 직원) 계정 예시**

     | 아이디 | 비밀번호 | 이름 | 권한 | 비고 |
     |---|---|---|---|---|
     | sadang | (원하는 비밀번호) | 사당점 | 매니저 | |

   **주의할 점**
   - `권한` 칸은 정확히 `관리자` 또는 `매니저`라고 입력해야 합니다.
   - 매니저 계정은 지점을 구분하지 않고 모두 같은 화면(매장 입력 항목)을
     봅니다. 매장별로 계정을 따로 만들어도 되고, 여러 매장이 같은 계정을
     함께 써도 됩니다.
   - 비밀번호는 평문으로 저장됩니다. 시트 공유 권한을 관계자 외에는 절대
     주지 마세요.

## 4. Vercel에 배포하기

1. https://vercel.com 에 로그인 후 **Add New > Project**로 이 저장소를
   가져옵니다.
2. **Root Directory**를 반드시 `insurance-intake`로 지정합니다. (crm-dashboard가
   아니라 이 폴더가 이 프로젝트의 Next.js 앱입니다.)
3. **Environment Variables**에 아래 5개를 등록합니다.

   | 이름 | 값 |
   |---|---|
   | `GOOGLE_SERVICE_ACCOUNT_EMAIL` | 1단계에서 복사한 서비스 계정 이메일 |
   | `GOOGLE_PRIVATE_KEY` | JSON 파일의 `private_key` 값 전체 (줄바꿈 포함 그대로 붙여넣기) |
   | `INTAKE_ADMIN_SPREADSHEET_ID` | 보험사업부 시트 URL의 `/d/`와 `/edit` 사이 문자열 |
   | `INTAKE_STORE_SPREADSHEET_ID` | 매장 시트 URL의 `/d/`와 `/edit` 사이 문자열 |
   | `JWT_SECRET` | 로그인 보안에 쓰이는 임의의 긴 문자열 (예: 32자 이상 랜덤 문자열) |

4. **Deploy**를 누르면 빌드가 시작되고, 끝나면 `https://프로젝트명.vercel.app`
   주소가 발급됩니다.
5. 발급된 주소로 접속해서 3단계에서 만든 계정으로 로그인해 봅니다.

> 로컬 컴퓨터에서 먼저 테스트해보고 싶다면 이 폴더 안에서 `.env.local.example`을
> 복사해 `.env.local`로 이름을 바꾸고 같은 값들을 채운 뒤 `npm install` →
> `npm run dev`를 실행하면 `http://localhost:3000`에서 확인할 수 있습니다.

## 5. 사용 방법

- **로그인**: 발급받은 주소 접속 → 계정관리 탭에 등록한 아이디/비밀번호로
  로그인합니다. 로그인은 로그인 시점으로부터 당일 자정까지 유효합니다.
- **관리자로 로그인하면**: 전체 항목(주민번호 포함)을 보고, 보험사업부
  진행상황 칼럼(중복보장점검완료일, 키트수령안내여부, 상담방식, 상담결과,
  상담상태, 거절사유분류, 보험가입여부/일자/보험료, 전환 인센티브 미적용
  등)과 매장 입력 칼럼(고객 선물 지급일자, 키트 불출인원, 특이사항)을 모두
  수정할 수 있습니다. 상단 "계정관리" 메뉴에서 계정별 로그인 실패횟수/잠김
  여부 확인, 비밀번호 초기화가 가능합니다.
- **매니저(매장)로 로그인하면**: 주민번호·생년월일·연락처 전체번호는 아예
  보이지 않고(연락처 뒤 4자리만 표시), 진행상황은 확인만 가능합니다.
  "고객 선물 지급일자 / 키트 불출인원 / 특이사항"만 직접 입력·수정할 수
  있습니다.
- **KPI 요약**: 화면 상단에 접수 건수, 중복보장점검 대기, 키트안내 대기,
  보험가입 완료(관리자는 즉시신청·거절 건수도 추가) 건수가 표시됩니다.
  검색/필터를 적용하면 그 조건에 맞는 건수로 바뀝니다.
- **검색/필터**: 이름·연락처 뒤 4자리·추천인·지점으로 검색하거나, 지점,
  상담상태, 중복보장점검 대기/완료, 키트수령안내 대기/완료로 좁혀볼 수
  있습니다.
- **설문 입력 칼럼(A~P열)이나 자동계산 칼럼은 절대 이 앱에서 수정하지
  않습니다.** 두 시트 모두 IMPORTRANGE·LOOKUP 등 수식이 걸린 칼럼이 섞여
  있어, 사람이 직접 입력해 온 칼럼(진행상황·매장 입력)만 정확히 골라서
  씁니다. 새 칼럼을 추가하거나 칼럼을 옮기면 `lib/sheetSchema.js`의
  `INTAKE_ADMIN_FIELDS` / `INTAKE_STORE_SHEET`에서 칼럼 위치를 함께
  맞춰줘야 합니다.

## 6. 문제 해결 (FAQ)

**Q. "INTAKE_ADMIN_SPREADSHEET_ID 환경변수가 설정되지 않았습니다" 오류가
나요.**
- Vercel(또는 `.env.local`)에 환경변수가 정확히 등록되어 있는지 확인하세요.
  등록/수정 후에는 Vercel에서 재배포(Redeploy)해야 반영됩니다.

**Q. 로그인이 안 돼요.**
- 보험사업부 시트에 `계정관리` 탭이 정확한 이름/칼럼 순서로 있는지, 서비스
  계정이 그 시트에 편집자로 공유되어 있는지 확인하세요.

**Q. 매장 입력 항목(고객 선물 지급일자 등)을 저장했는데 다른 사람 화면에는
안 보여요.**
- 매장 시트의 IMPORTRANGE가 반영되기까지 몇 초~1분 정도 걸릴 수 있습니다.
  저장한 화면에는 즉시 반영되어 보이고, 다른 사람 화면은 새로고침하면
  최신값을 다시 받아옵니다.

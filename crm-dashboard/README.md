# AJF 회원 관리 & 컨택 히스토리 대시보드

구글 시트(관리자 종합 시트 + 매니저별 개별 시트)를 데이터 원본으로 사용하는
회원 관리 웹 대시보드입니다. 매니저는 본인이 담당하는 회원만 보고 수정할 수 있고,
관리자는 전체 회원을 보고 수정할 수 있습니다. 웹에서 수정한 내용은 구글 시트
(관리자 시트 + 해당 매니저 개별 시트)에 자동으로 반영됩니다.

이 문서는 **코딩을 전혀 모르는 분도 그대로 따라 하면 서비스를 띄울 수 있도록**
순서대로 작성했습니다. 위에서부터 차례대로 진행해 주세요.

---

## 0. 준비물 체크리스트

- [ ] 구글 계정 (시트 소유자 계정)
- [ ] 관리자 종합 시트 (관리_컨택종합리스트) 구글 시트 파일
- [ ] 매니저별 개별 시트 파일들 (M매니저명 형태)
- [ ] GitHub 계정 (이 코드가 들어있는 저장소에 접근 가능해야 함)
- [ ] Vercel 계정 (무료 플랜으로 충분, GitHub 계정으로 가입 가능)

## 1. 구글 클라우드 서비스 계정 만들기

웹 대시보드가 사람 대신 "서비스 계정"이라는 전용 로봇 계정으로 구글 시트를
읽고 쓰게 됩니다. 아래 순서대로 진행하세요.

1. https://console.cloud.google.com/ 접속 후 구글 계정으로 로그인합니다.
2. 상단의 프로젝트 선택 메뉴에서 **새 프로젝트**를 만듭니다. (이름은 자유롭게, 예: `ajf-crm`)
3. 좌측 메뉴 또는 검색창에서 **"API 및 서비스" > "라이브러리"**로 이동합니다.
   - **Google Sheets API**를 검색해서 **사용(Enable)**으로 설정합니다.
   - **Google Drive API**도 검색해서 **사용(Enable)**으로 설정합니다.
4. 좌측 메뉴 **"API 및 서비스" > "사용자 인증 정보"**로 이동합니다.
5. 상단 **"+ 사용자 인증 정보 만들기" > "서비스 계정"**을 클릭합니다.
   - 서비스 계정 이름: 자유롭게 입력 (예: `ajf-crm-bot`)
   - 나머지 단계는 기본값으로 두고 **완료**를 누릅니다.
6. 생성된 서비스 계정 목록에서 방금 만든 계정을 클릭합니다.
7. 상단 탭에서 **"키(Keys)"** 탭으로 이동 후 **"키 추가" > "새 키 만들기"**를 클릭합니다.
   - 키 형식은 **JSON**을 선택하고 **만들기**를 누르면 JSON 파일이 자동으로 다운로드됩니다.
   - **이 JSON 파일을 안전하게 보관하세요. 외부에 노출되면 시트에 누구나 접근할 수 있습니다.**
8. 서비스 계정 상세 화면 상단에 보이는 **이메일 주소**를 복사해 둡니다.
   (`xxxxx@xxxxx.iam.gserviceaccount.com` 형태입니다. 이후 환경변수 설정에 사용합니다.)
9. 다운로드한 JSON 파일을 열어 보면 다음과 같은 항목이 있습니다. 이후 설정에 필요합니다.
   - `"client_email"` → 8번에서 복사한 이메일과 동일
   - `"private_key"` → `-----BEGIN PRIVATE KEY-----`로 시작하는 긴 문자열 전체

## 2. 구글 시트를 서비스 계정과 공유하기

서비스 계정은 "사람"이 아니므로, 시트에 직접 **공유(Share)**를 해줘야 접근할 수 있습니다.

1. **관리자 종합 시트**를 열고 우측 상단 **공유** 버튼을 누릅니다.
2. 1단계 8번에서 복사한 서비스 계정 이메일을 추가하고 권한을 **편집자(Editor)**로 설정한 뒤 공유합니다.
3. **모든 매니저 개별 시트**도 각각 동일하게 서비스 계정 이메일을 편집자로 공유합니다.

> 💡 **모든 매니저 시트가 같은 공유 드라이브(Shared Drive) 안에 있다면**, 시트 하나하나
> 공유하는 대신 그 공유 드라이브 자체에 서비스 계정을 멤버로 추가(콘텐츠 관리자 이상 권한)하면
> 한 번에 모든 현재/향후 매니저 시트에 접근 권한이 부여되어 더 편리합니다.
> (공유 드라이브 우측 상단 드라이브 이름 클릭 → "회원 관리" → 이메일 추가)

## 3. 관리자 시트에 "계정관리" 탭 만들기

로그인 계정 정보와, 매니저 이름별로 어떤 개별 시트를 봐야 하는지를 관리자 시트
안의 새 탭에서 직접 관리합니다. 이렇게 하면 코드 수정 없이 시트에서 매니저를
추가/삭제하거나 비밀번호를 바꿀 수 있습니다.

1. **관리자 종합 시트** 하단에 새 탭을 추가하고 이름을 정확히 `계정관리`로 지정합니다.
2. 1행(헤더)에 아래 8개 컬럼명을 **A1부터 순서대로** 입력합니다. (`실패횟수`, `잠김여부`는
   로그인 5회 실패시 자동 잠금 기능에 쓰이는 컬럼이며, 앱이 자동으로 값을 채우므로
   직접 입력할 필요는 없습니다. 헤더만 만들어두면 시트에서도 한눈에 보입니다.)

   | A | B | C | D | E | F | G | H |
   |---|---|---|---|---|---|---|---|
   | 아이디 | 비밀번호 | 이름 | 권한 | 개인시트URL | 비고 | 실패횟수 | 잠김여부 |

3. 2행부터 계정을 한 줄씩 입력합니다. (G, H열은 비워두면 앱이 자동으로 0/빈칸으로 채웁니다.)
   - **관리자 계정 예시**

     | 아이디 | 비밀번호 | 이름 | 권한 | 개인시트URL | 비고 |
     |---|---|---|---|---|---|
     | admin | (원하는 비밀번호) | 관리자 | 관리자 | (비워둠) | |

   - **매니저 계정 예시** (최현정 매니저)

     | 아이디 | 비밀번호 | 이름 | 권한 | 개인시트URL | 비고 |
     |---|---|---|---|---|---|
     | choihj | (원하는 비밀번호) | 최현정 | 매니저 | https://docs.google.com/spreadsheets/d/.../edit | |

   **주의할 점**
   - `권한` 칸은 정확히 `관리자` 또는 `매니저`라고 입력해야 합니다.
   - `이름` 칸은 관리자 종합 시트의 `담당매니저` 컬럼에 적힌 이름과 **글자가 정확히 일치**해야
     합니다(공백 포함). 이름이 다르면 그 매니저는 자기 데이터를 볼 수 없습니다.
   - `개인시트URL`은 그 매니저의 개별 구글 시트 주소를 전체 그대로 붙여넣으면 됩니다
     (`/edit?gid=...` 가 붙어있어도 무관합니다).
   - 매니저가 늘어나면 이 탭에 한 줄씩 계속 추가하면 됩니다. 코드 수정이 필요 없습니다.
   - 비밀번호는 평문(그대로 보이는 글자)으로 저장됩니다. 시트 자체의 공유 권한을
     관계자 외에는 절대 주지 않도록 주의해 주세요. (로그인한 사용자는 화면에서
     본인 비밀번호를 직접 바꿀 수 있고, 관리자는 계정관리 화면에서 누구의 비밀번호든
     초기화할 수 있습니다. 자세한 내용은 6번 항목 참고)
   - **이미 계정관리 탭을 운영 중이라면**: G1, H1에 `실패횟수`, `잠김여부` 헤더만
     추가해주면 됩니다. 기존 계정 데이터(A~F열)는 그대로 두어도 됩니다.

## 4. 데이터 탭 자동 인식 방식 (참고)

관리자 시트와 매니저 개별 시트는 탭 이름이 서로 달라도 됩니다. 이 앱은 각
스프레드시트를 열어서, 헤더 행에 `연락처`와 `담당매니저` 컬럼이 동시에 있는
탭을 자동으로 "데이터 탭"으로 인식합니다. 따라서 탭 이름을 특정 이름으로
맞출 필요는 없지만, **헤더(1행)에 `연락처`, `담당매니저` 컬럼명은 정확히
존재해야** 합니다.

컨택 히스토리 컬럼명은 관리자 시트(`컨택 히스토리(매니저)`)와 매니저 시트
(`컨택 히스토리`)가 달라도 자동으로 같은 항목으로 인식되도록 처리되어 있습니다.
그 외 컬럼명은 아래와 동일하게 맞춰주세요.

```
매니저 시트 바로가기, 그룹, 브랜드, 광역/인스타, 권역, 이름, 연락처,
지점/대리점 명, 담당매니저, 배분일자, 우선컨택 딜러여부, 고효율딜러여부,
고효율딜러수치, 컨택여부, 최초컨택일자, 재컨택일자, 문자여부, 컨택 호의도,
컨택 히스토리(매니저), 사전예약여부, App가입일자, 누적 계약체결건수,
직전 60일 계약체결건수, 직전 1년 본인 10% 횟수, 관리자 특이사항
```

(매니저 개별 시트는 A열이 `매니저 시트 바로가기` 대신 `seq`인 것만 다르고 나머지는 동일합니다.)

## 5. Vercel에 배포하기

1. 이 저장소(GitHub)를 https://vercel.com 에 로그인 후 **Add New > Project**로
   가져옵니다. (GitHub 계정 연동이 안 되어 있다면 화면 안내에 따라 연동합니다.)
2. **Root Directory**를 반드시 `crm-dashboard`로 지정합니다. (저장소 최상위가
   아니라 이 폴더가 Next.js 프로젝트입니다.)
3. **Environment Variables**에 아래 4개를 등록합니다.

   | 이름 | 값 |
   |---|---|
   | `GOOGLE_SERVICE_ACCOUNT_EMAIL` | 1단계 8번에서 복사한 서비스 계정 이메일 |
   | `GOOGLE_PRIVATE_KEY` | JSON 파일의 `private_key` 값 전체 (줄바꿈 포함 그대로 붙여넣기) |
   | `ADMIN_SPREADSHEET_ID` | 관리자 종합 시트 URL의 `/d/`와 `/edit` 사이 긴 문자열 |
   | `JWT_SECRET` | 로그인 보안에 쓰이는 임의의 긴 문자열 (예: 32자 이상 랜덤 문자열) |

   - `GOOGLE_PRIVATE_KEY`를 붙여넣을 때 Vercel 입력창이 줄바꿈을 그대로
     받아주므로 JSON 파일 안의 값을 그대로(따옴표 제외) 복사해서 넣으면 됩니다.
   - `ADMIN_SPREADSHEET_ID` 예시: 시트 주소가
     `https://docs.google.com/spreadsheets/d/1QYmS11e-oUON1RwXIh8RgLYAQKC6Ev6Ge3Rz6nps40k/edit?gid=...`
     라면 `1QYmS11e-oUON1RwXIh8RgLYAQKC6Ev6Ge3Rz6nps40k` 부분만 넣습니다.
4. **Deploy**를 누르면 빌드가 시작되고, 끝나면 `https://프로젝트명.vercel.app` 주소가 발급됩니다.
5. 발급된 주소로 접속해서 3단계에서 만든 계정으로 로그인해 봅니다.

> 로컬 컴퓨터에서 먼저 테스트해보고 싶다면 `crm-dashboard` 폴더 안에
> `.env.local.example`을 복사해 `.env.local`로 이름을 바꾸고 같은 값들을 채운 뒤
> `npm install` → `npm run dev`를 실행하면 `http://localhost:3000`에서 확인할 수 있습니다.

## 6. 사용 방법

- **로그인**: 발급받은 주소 접속 → 계정관리 탭에 등록한 아이디/비밀번호로 로그인합니다.
  로그인은 **로그인한 시점으로부터 24시간 동안** 유효하며, 24시간이 지나면 자동으로
  로그아웃되어 다시 로그인해야 합니다.
- **비밀번호 5회 오류시 자동 잠금**: 같은 계정으로 비밀번호를 5번 연속 틀리면 계정이
  잠기고, 비밀번호가 맞아도 더 이상 로그인할 수 없습니다. 잠긴 계정은 **관리자가
  계정관리 화면에서 비밀번호를 초기화**해야 다시 로그인할 수 있습니다.
- **본인 비밀번호 변경**: 로그인 후 화면 우측 상단의 "비밀번호 변경" 버튼으로 누구나
  본인 비밀번호를 직접 바꿀 수 있습니다. 현재 비밀번호를 알아야 변경할 수 있습니다.
- **매니저로 로그인하면**: 본인이 `담당매니저`로 지정된 회원만 표 형태로 보입니다.
  다른 매니저의 회원, `관리자 특이사항`, `매니저 시트 바로가기` 컬럼은 보이지 않습니다.
- **관리자로 로그인하면**: 전체 회원이 보이고, 매니저 필터로 특정 매니저만 골라볼 수
  있으며, 상단 "구글 시트 원본 열기" 버튼으로 원본 관리자 시트를 바로 열 수 있습니다.
  화면 상단 "계정관리" 탭에서는 전체 계정의 로그인 실패횟수/잠김 여부를 확인하고,
  각 계정의 "비밀번호 초기화" 버튼으로 새 비밀번호를 직접 지정해줄 수 있습니다.
  초기화하면 잠금도 함께 풀리고 실패횟수도 0으로 초기화됩니다.
- **검색/필터**: 상단 필터 카드에서 이름·연락처·지점으로 검색하거나, 컨택여부/사전예약여부로
  좁혀볼 수 있습니다. (관리자는 매니저 필터도 추가로 사용 가능)
- **편집**: 표의 행을 클릭하면 편집 창이 열립니다.
  - 매니저는 컨택여부, 최초컨택일자, 재컨택일자, 문자여부, 컨택 호의도, 컨택 히스토리,
    사전예약여부를 수정할 수 있습니다.
  - 관리자는 위 항목에 더해 그룹/브랜드/지점/담당매니저/배분일자 등 모든 항목을 수정할 수 있습니다.
  - 저장하면 관리자 종합 시트와, 해당 회원의 담당매니저 개별 시트 양쪽에 반영됩니다.
    개별 시트에 그 회원 행이 아직 없다면 새 행으로 추가됩니다. 두 시트는 완전히
    별도 파일로 독립 관리되므로, 이미 값이 달랐던 기존 데이터까지 자동으로 맞춰주지는
    않으며 **이 앱을 통해 저장한 시점 이후의 수정 내용만** 양쪽에 동기화됩니다.
- **CSV 내보내기**: 현재 필터링된 목록을 엑셀에서 바로 열 수 있는 CSV 파일로 내려받을 수 있습니다.
- **로그아웃**: 우측 상단 로그아웃 버튼.

## 7. 문제 해결 (FAQ)

**Q. 로그인이 안 돼요.**
- 계정관리 탭의 아이디/비밀번호 칸에 앞뒤 공백이 들어가지 않았는지 확인하세요.
- `권한` 칸이 정확히 `관리자` 또는 `매니저`로 적혀 있는지 확인하세요.

**Q. 매니저로 로그인했는데 회원이 하나도 안 보여요.**
- 계정관리 탭의 `이름` 칸이 관리자 종합 시트의 `담당매니저` 컬럼 값과 **정확히** 일치하는지
  확인하세요 (오탈자, 공백, 띄어쓰기 포함).

**Q. 수정했는데 매니저 개별 시트에는 반영이 안 됐다고 나와요.**
- 편집 후 뜨는 안내 메시지를 확인하세요. 보통 다음 두 경우입니다.
  1. 계정관리 탭에 그 매니저의 `개인시트URL`이 비어 있는 경우 → URL을 채워주세요.
  2. 그 회원이 아직 담당매니저로 지정되지 않은 경우.

**Q. "스프레드시트에서 회원 데이터 탭을 찾지 못했습니다" 오류가 나요.**
- 해당 시트의 어떤 탭에도 헤더 행에 `연락처`와 `담당매니저`가 동시에 들어있지 않다는
  뜻입니다. 두 헤더가 정확한 문구로 1행에 들어있는지 확인하세요.

**Q. "비밀번호를 5회 이상 잘못 입력하여 계정이 잠겼습니다"라고 나와요.**
- 보안을 위한 자동 잠금 기능입니다. 관리자 계정으로 로그인 → 상단 "계정관리" 탭 →
  해당 계정의 "비밀번호 초기화" 버튼으로 새 비밀번호를 지정하면 잠금이 풀리고
  바로 로그인할 수 있습니다. (본인 계정이 잠긴 경우 다른 관리자에게 요청하세요.)

**Q. 서비스 계정 권한 오류(403/PERMISSION_DENIED)가 나요.**
- 2단계(시트 공유)를 다시 확인하세요. 새로 추가된 매니저 시트를 공유하지 않고
  계정관리 탭에만 URL을 적어둔 경우 자주 발생합니다.

**Q. 비밀번호를 더 안전하게 관리하고 싶어요.**
- 현재는 소규모 내부용 도구로 빠르게 운영할 수 있도록 시트에 평문으로 비밀번호를
  저장하는 방식을 쓰고 있습니다. 시트 자체의 공유 대상을 최소 인원으로 유지하고,
  주기적으로 비밀번호를 바꿔주는 것을 권장합니다.

## 8. 알려진 보안 권고 사항 (참고)

`npm audit` 기준으로 Next.js 14 계열 일부 취약점 권고가 남아있으나, 모두
이미지 최적화(`next/image`), App Router/Server Components, 커스텀
미들웨어, `i18n` 리라이트 기능과 관련된 것이며 이 프로젝트는 해당 기능을
전혀 사용하지 않는 단순 Pages Router 앱입니다. 정기적으로
`npm outdated next` 로 새 패치 버전이 나왔는지 확인해 업그레이드하는 것을
권장합니다.
